# AI Engineering Workflow

## Summary

A modern development pattern prioritizing product realization over sequential model training. The workflow shifts from a linear, data-centric approach to an iterative, product-centric approach.

## Core Idea

Traditional ML engineering follows a linear path (Data → Model → Product), where product development is often an afterthought. AI engineering, enabled by readily available foundation models, reverses this, allowing engineers to build a Minimum Viable Product (MVP) first, and only invest in data and model refinement once product viability is demonstrated (Product → Data → Model).

## Practical Use

When initiating a new AI project, use this framework to scope the initial MVP. Focus on defining the user experience and core functionality first. Treat data and model selection as iterative inputs that validate the product concept, rather than prerequisites.
