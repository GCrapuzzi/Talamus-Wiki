# Hallucination Detection and Mitigation

## Summary

The process of identifying and correcting factually incorrect or unsupported outputs generated by LLMs.

## Core Idea

Hallucination occurs because models can generate plausible but non-existent information. Mitigation requires moving beyond simple prompt engineering to structural changes in the model's training or inference process.

## Practical Use

1. **Pre-Check:** Implement a validation layer that checks generated claims against a trusted knowledge base (RAG). 2. **Training:** Utilize Reinforcement Learning (RL) and Supervised Fine-Tuning (SFT) with factual/counterfactual signals. 3. **Prompting:** Explicitly instruct the model to cite sources and refuse to answer if information is missing.
