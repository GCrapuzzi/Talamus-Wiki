# AI Productivity Curve in Software Engineering

## Summary

A framework describing the differential productivity gains of AI across various software engineering tasks.

## Core Idea

AI provides the highest productivity gains (e.g., 2x for documentation, 25-50% for refactoring) for tasks that are repetitive, structured, or involve pattern application. Productivity gains diminish significantly for highly complex, novel, or deeply architectural tasks.

## Practical Use

Use this framework for resource allocation: Estimate the bulk of the effort for simple, repetitive tasks using AI multipliers. Reserve human engineering time for the core, complex logic that requires deep domain knowledge and novel problem-solving.
