# System Isolation and Sandboxing

## Summary

Executing untrusted or generated code (e.g., code generated by the LLM) within a restricted, isolated environment (like a Virtual Machine or sandbox) that cannot affect the host system or user's main machine.

## Core Idea

This defense mechanism limits the blast radius of malicious code injection. If the code attempts harmful actions (like installing malware), the damage is contained entirely within the isolated environment.

## Practical Use

If your AI system generates code that interacts with the operating system or external resources, *never* execute it directly. Always pass it through a sandboxed execution layer.
