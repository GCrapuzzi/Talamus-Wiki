# Data Flywheel

## Summary

A system design pattern where data generated by users interacting with an application is captured, annotated, and subsequently used to retrain and improve the AI model, leading to better user experience and increased adoption.

## Core Idea

The most valuable data is application data because it is perfectly relevant and aligned with the specific task distribution the model needs to learn. This creates a self-reinforcing cycle of data generation and model improvement.

## Practical Use

Design user interaction points to maximize data capture (e.g., logging user edits, capturing user feedback, tracking system usage). Implement a feedback mechanism that routes this raw, high-relevance data back into the annotation and fine-tuning pipeline.
