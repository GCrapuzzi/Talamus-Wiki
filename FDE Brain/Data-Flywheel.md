# Data Flywheel

A self-reinforcing loop where user-generated application data continuously improves the model, which in turn improves the product, attracting more users and data.

Application data is ideal because it inherently matches the target distribution—something extremely hard to achieve with external data sources. User-generated data includes:

- **User content** — direct inputs and outputs
- **System-generated usage data** — interaction logs, latency, error rates
- **User feedback** — explicit ratings, corrections, preferences

The data flywheel is a durable competitive advantage: once spinning, competitors can't replicate your dataset even if they replicate your model.
