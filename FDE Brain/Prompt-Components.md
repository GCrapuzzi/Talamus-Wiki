# Prompt Components

## Summary

The foundational parts that constitute a complete and effective prompt: System, User, Examples, and Context.

## Core Idea

A prompt is not a single block of text; it is a structured communication. Understanding these components allows an engineer to systematically improve prompt quality and reliability.

## Practical Use

When designing a complex application, structure the prompt using these four elements: 1. **System:** Defines the model's role/rules. 2. **User:** Contains the specific query. 3. **Examples:** Provides few-shot demonstrations. 4. **Context:** Supplies necessary background information.
