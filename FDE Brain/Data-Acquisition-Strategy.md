# Data Acquisition Strategy

## Summary

A structured approach to building a dataset by balancing budget, quality, diversity, and compliance across multiple sources (public, proprietary, user-generated, synthetic).

## Core Idea

Effective data acquisition is not just about quantity, but about achieving the necessary quality, diversity, and relevance while managing the trade-offs between data annotation costs and compute resources. The ideal strategy leverages proprietary, user-generated data to create a continuous improvement loop (data flywheel).

## Practical Use

When starting a new AI project, map out the data lifecycle: 1. Identify the primary data source (ideally user-generated application data). 2. Estimate the budget constraints (data annotation vs. compute). 3. Design a feedback loop (data flywheel) to ensure continuous, relevant data input. 4. Plan for a mixed-method approach combining multiple sources.
