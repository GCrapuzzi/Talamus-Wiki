# Reranking Mechanism

## Summary

A post-retrieval step where a dedicated, often more computationally expensive, model is used to re-score and re-order the initial set of candidate documents provided by the primary retriever. This increases the precision of the final context.

## Core Idea

Reranking acts as a quality filter. It is essential when the initial retriever is broad (high recall, lower precision) and the context window is limited, ensuring that only the absolute best documents are passed to the LLM.

## Practical Use

After retrieving the top 50 documents using vector search, pass those 50 documents through a specialized cross-encoder model (the reranker) to get a refined score, and then only pass the top 5 to the LLM.
